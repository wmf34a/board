네이밍 규약

기본 규칙

공통 규칙

  • 첫 시작에 숫자, 특수문자, 대문자의 사용은 지양한다.
  • '형태(역할)_의미_상태' 순서로 조합하며, 3단계를 넘어가지 않도록 권장한다.
    예) btn_apply_on, box_news, box_reply_open 등
  • 구분 기준이 모호하거나 추가 분량 예측이 불가한 경우 숫자를 붙여 구분짓는다.
    넘버링시 파일명 마지막단계 끝에 붙이되, 자릿수를 맞추기 위한 숫자 0은 쓰지 않는다.
    예) bul_dot_white1, bul_dot_white2, bul_dot_white3
  • naming 정의 시 prefix(접두어), subfix(접두 보조), suffix(접미어)를 최대한 활용한다.

이미지 네이밍 규칙

  • 일반 이미지의 prefix는 img_로 한다.
  • 스프라이트 이미지의 prefix는 sp_로 한다.

id/class 규칙

  • id, class 모두 underscore(_) 방식으로 사용한다.
  • id는 화면에서의 고유 기능을, class는 요소 기능을 표현하도록 naming한다.
  • id는 문서 내 한 번만 사용하며, id, class naming은 가급적 동일하지 않게 naming 한다.

폴더 및 파일명 규칙

폴더 및 HTML 파일명 지정은 네이밍 기본 규칙에서 예외로 하며, 다음과 같이 관리한다.

파일 및 폴더 구조 보기

html/ 하위폴더 네이밍

  • 정보구조도(IA)에 정의된 메뉴구조에 의해, 대메뉴(1뎁스)를 폴더별로 구성한다.
  • 페이지 분량이 방대할 경우, 필요에 따라 하위뎁스를 하위폴더로 둘 수 있다.
  • 대메뉴명을 폴더명으로 하며, 메뉴명이 한글일 경우 한글로 정의한다.
  • 특수문자는 "(", ")" 만 허용하며, 공백(띄어쓰기)을 허용하지 않는다.

images/content/ 하위폴더 네이밍

  • 이미지 분량이 적은 경우 하위폴더를 생략할 수 있다.
  • 정보구조도(IA)에 정의된 메뉴구조에 의해, 대메뉴(1뎁스)를 폴더별로 구성한다.
  • 대메뉴(1뎁스)명을 폴더명으로 하며, 축약형 영문으로 정의한다.
  • 특수문자는 "(", ")" 만 허용하며, 공백(띄어쓰기)을 허용하지 않는다.

HTML 파일 네이밍

  • 정보구조도(IA)에 정의된 메뉴구조에 의해 한글로 정의한다.
  • 특수문자는 "(", ")", "_" 만 허용하며, 공백(띄어쓰기)을 허용하지 않는다.
  • 폴더에 대메뉴는 정의되어 있으므로, 중/소메뉴부터 언더바(_)로 구분하여 파일명을 정의한다.
    예) 중메뉴_소메뉴_화면명.html
  • 팝업의 경우, 화면명 앞에 (팝업)이라 표시한다.
    예) 중메뉴_소메뉴_(팝업)화면명.html
