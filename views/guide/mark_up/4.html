주석 표기

공통 규칙

모호하고 장황한 문장을 지양하여, 전달코자하는 핵심내용만 명료하고 정확한 명사형으로 표기한다.

<!-- 개인회원이 로그인했을 때 보입니다. -->	(X) 장황한 문장
<!-- 개인회원 로그인시 노출 -->			(O)

주석을 여닫는 코드와 주석 사이에 공백이 없으면 간헐적으로 오류가 발생하는 문제가 있으므로,
<!--(공백)주석내용(공백)--> 과 같이 한 칸 씩 띄어쓴다.

<!--주석내용-->		(X) 문구와 주석 여닫는 코드 사이를 띄어쓰지 않음
<!-- 주석내용 -->	(O)

적용 범위의 시작과 끝에 동일한 문구로 기입하되, 닫는 주석에는 주석 앞에 슬래시 두개+공백(// )을 붙인다.

<!-- 개인회원일 때 노출 (시작) -->
	...내용...
<!-- 개인회원일 때 노출 (끝) -->		(X) 여닫는 주석 문구가 동일하지 않음

<!-- 개인회원일 때 노출 -->
	...내용...
<!-- //개인회원일 때 노출 -->		(X) 슬래시(//) 와 문구 사이는 띄어써야 함

<!-- 개인회원일 때 노출 -->
	...내용...
<!-- // 개인회원일 때 노출 -->		(O)

주석의 내용이 길어질 경우

여는 주석의 첫 줄은 닫는 주석과 동일한 문구로 요약하며,
여는 주석을 개행하여 두 번 째 줄부터 전달사항을 자유롭게 기입한다. 

<!-- 개인회원일 때 결과목록
	개인회원이 ...중략... 보여줍니다.
-->
	...내용...
<!-- // 개인회원일 때 결과목록 -->

주석 적용 범위가 매우 작은 경우

코드 한 줄에 대한 주석일 경우, 닫는 태그의 뒤쪽에 닫는 주석으로 기입한다.

<th scope="col">수량</th><!-- // 구매자일 때 노출 -->

그루핑 종료 주석

레이아웃 컨테이너의 닫는 태그 뒤에는 주석을 기입하지 않는다.
이밖에 코드가 매우 길어 섹션 구분에 혼선 가능성이 있다고 판단될 시에는 그루핑 종료 주석을 활용할 수 있다. 

<div class="tab_cont">
	...내용...
</div><!-- // .tab_cont -->

업데이트 주석

SVN등의 버전 관리 시스템이 구축되지 않은 개발환경일 경우,
<!-- YYYY.MM.DD : 주석내용 --> 형태로 업데이트 날짜를 함께 기입한다. 

<!-- 2014/00/00 : 삭제버튼 추가 -->
	...내용...
<!-- // 2014/00/00 : 삭제버튼 추가 -->	(X) 날짜표기형식이 YYYY.MM.DD가 아님

<!-- 2014.00.00 : 삭제버튼 추가 -->
	...내용...
<!-- // 2014.00.00 : 삭제버튼 추가 -->	(O)

퍼블리셔 전달용 주석

대체로 주석은 개발자를 위한 내용이지만, 여러명의 퍼블리셔가 함께 작업하는 경우 커뮤니케이션을 위해 전용 주석을 사용할 수 있다. 이 때 혼동을 막기 위해 주석내용 앞에 접두사 [P]를 붙이며, 퍼블리싱 후 삭제토록 한다. 

<!-- [P] 주석내용 -->
<!-- // [P] 주석내용 -->
<!-- 2014.00.00 : [P] 주석내용 -->
<!-- // 2014.00.00 : [P] 주석내용 -->
